# index-web-archive

## 项目简介

`index-web-archive` 是一个用于归档和发布多个独立 HTML 页面的仓库。本项目不针对任何单个域名或具体网站，其核心目的是收集、保存并提供一系列静态 HTML 页面的访问入口，方便对这些页面进行集中管理和历史查看。

## 目录结构

```
index-web-archive/
├── pages/               # 存放所有归档的 HTML 页面
│   ├── page-01.html
│   ├── page-02.html
│   └── ...
├── assets/              # 页面引用的公共资源（CSS、JS、图片等）
│   ├── styles/
│   ├── scripts/
│   └── images/
├── index.html           # 根目录下的入口页面，提供归档页面的导航列表
└── README.md            # 本文件
```

- `pages/`：每个独立 HTML 页面存放在此目录下，文件名遵循一定的命名规范（如 `page-序号.html`），便于识别和排序。
- `assets/`：存放页面所需的公共资源文件，按类型分目录组织。
- `index.html`：仓库的首页，通常包含一个简单的列表或网格，展示所有归档页面的标题和链接。

## 页面归档说明

本仓库中的每个 HTML 页面均来自不同的来源，它们被收集并归档于此，以保留其原始内容与结构。归档页面一般遵循以下原则：

- **完整性**：尽量保持页面内容的完整性，包括样式、脚本及图片等资源。
- **独立性**：每个页面自包含，不依赖外部在线资源（除非必要），以确保离线或长期可访问。
- **版本记录**：每次新增或更新页面时，会通过 Git 提交记录变更，方便追溯历史版本。

## 维护说明

- **添加新页面**：将新的 HTML 文件放入 `pages/` 目录，并更新 `index.html` 中的导航列表。
- **更新已有页面**：替换 `pages/` 下的对应文件，并提交变更说明。
- **资源管理**：如需添加公共资源，请放置在 `assets/` 对应子目录下，并在页面中正确引用。
- **提交规范**：建议提交信息简要描述本次操作，例如“添加页面：xxx”或“更新页面：xxx”。

## 注意事项

- 本仓库仅用于归档目的，不对页面内容做额外修改或筛选。
- 所有页面均以静态形式保存，不涉及动态交互或后端服务。
- 如需引用或分发归档页面，请遵守原始内容的版权规定。
